# HomeProxy 备忘：Client Settings vs Server Settings

本文用于快速解释：在 HomeProxy UI 里，`Client Settings` 和 `Server Settings` 分别是做什么的、适合谁用、常见误解是什么。

---

## 1. Client Settings（客户端设置）是干什么的

`Client Settings` 面向“路由器作为出站代理客户端”的场景，核心目标是：

- 让路由器和内网设备通过你配置的代理节点出网
- 做透明代理/分流（例如大陆直连、国外走代理）
- 管理 DNS 与路由规则

用户视角常见用途：

1. 配置主节点（`main_node`）让家庭网络开始走代理
2. 选择路由模式（如绕过大陆、全局、自定义）
3. 调整 DNS（国外 DNS、中国 DNS、策略、缓存）
4. 配合访问控制（ACL）控制哪些设备/流量走代理

一句话：  
`Client Settings` 是“我家设备怎么通过代理上网”。

---

## 2. Server Settings（服务端设置）是干什么的

`Server Settings` 面向“路由器作为代理服务端”的场景，核心目标是：

- 在路由器上开启入站服务（例如让外部客户端连回家里路由器）
- 配置服务端协议、证书、监听等参数

用户视角常见用途：

1. 想把家里路由器当作一个可被外部连接的代理入口
2. 需要给外部设备/远程环境提供回家隧道或代理入口

一句话：  
`Server Settings` 是“别人怎么连到我这台路由器上的代理服务”。

---

## 3. 两者关系与启动逻辑（高频疑问）

- 两者是独立能力，可以只用其中一个。
- 只配置 `Client Settings`（并选好主节点）就可以让家庭透明代理生效，不需要开 `Server Settings`。
- `Save & Apply` 会提交并应用配置，但是否真正拉起核心实例，取决于配置是否满足启动条件（例如客户端有有效主节点，或服务端被启用）。

---

## 4. 快速选型

- 你的目标是“家里设备科学上网/分流”：用 `Client Settings`
- 你的目标是“让外部设备连回你路由器做代理”：用 `Server Settings`
- 两个都需要：可以同时配置

---

## 5. 给用户的一句解释模板

> Client 是“我出去”，Server 是“别人进来”。  
> 大多数家庭代理用户只需要配置 Client。
